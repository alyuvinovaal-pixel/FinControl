## 📝 ОБЪЯСНЕНИЯ

**TG-DB — Обновить БД** — бот подключён к старой схеме, в которой нет новых полей (`currency`, `status`, `notifications_enabled` и т.д.). Нужно переподключиться к актуальной `database.db` и использовать функции из `db_queries.py` вместо прямых SQL-запросов. Не дублировать логику — только импортировать из общего модуля.

Структура проекта должна быть такой:
```
fincontrolapp/
├── database.py      ← общая схема
├── db_queries.py    ← общие запросы
└── bot/
    └── bot.py       ← импортирует db_queries
```

**TG-2 + TG-3 — Привязка аккаунта** — в настройках приложения пользователю показывается ссылка `https://t.me/ИМЯ_БОТА?start=USER_ID`. Бот принимает `user_id` из параметра команды `/start` и сохраняет `chat_id` этого пользователя в поле `telegram_id` таблицы `users`. После этого бот знает кому отправлять уведомления.

**TG-4 — Ввод транзакций** — пользователь пишет `+5000 зарплата` или `-300 кафе`. Бот парсит знак (доход/расход), сумму и описание. Если описание совпадает с известной категорией — использует её, иначе «Другое». Вызывает `add_transaction()` из `db_queries.py` и отвечает подтверждением.

**TG-5 + TG-6 — Уведомления** — нужен планировщик (`APScheduler`), который запускается вместе с ботом и каждый день в нужное время проверяет БД. Для подписок: если до `charge_day` ≤ 3 дней — отправить сообщение. Для целей: отправлять в день по расписанию если задана периодичность. Уведомления отправляются только пользователям у которых `notifications_enabled = 1`.

**TG-7 — /stats** — получить `user_id` по `chat_id`, вызвать `get_monthly_balance()` и `get_balance()` из `db_queries.py`, отформатировать и отправить. Добавить кнопки «Неделя / Месяц / Год» для выбора периода.

**TG-9 — Архитектура** — для локальной разработки и тестирования достаточно long polling (проще запустить). Webhook нужен только если бот будет работать на сервере. Выбор зависит от того, где будет деплой.

**TG-8 — Авторизация** — использовать готовую функцию `verify_password()` из `pages/auth.py`, не писать заново. Бот запрашивает email/телефон и пароль, проверяет через ту же логику что и приложение.
