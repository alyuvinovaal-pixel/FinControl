## 📝 ОБЪЯСНЕНИЯ

**Общее** — работа разделена на две части: сначала подключить реальные данные к уже существующей странице аналитики (до 8 апреля), потом написать ML-модели (после 25 апреля).

**ANAL-1 — Реальные данные** — в `analytics.py` сейчас в начале файла стоят захардкоженные константы `MONTHLY_DATA` и `EXPENSE_CATEGORIES`. Их нужно убрать и в методе `build_body()` вызывать `get_monthly_data(user_id)` и `get_expense_breakdown(user_id)` из `db_queries.py`. Если данных нет — показывать заглушку «Недостаточно данных» вместо пустого графика.

**ANAL-4 — flet.charts** — текущие графики нарисованы вручную через `ft.Stack` с абсолютным позиционированием. Это хрупко и выглядит неточно. В Flet есть встроенные `fltc.BarChart` и `fltc.LineChart` — заменить на них. После замены Designer настроит внешний вид (цвета, шрифты, tooltip).

**SUB-3 — Пауза подписок** — добавить две функции в `db_queries.py`: `pause_subscription(sub_id)` и `resume_subscription(sub_id)`. Они меняют поле `is_paused` (которое Team Lead добавит через миграцию). В UI кнопка «Удалить» заменяется на меню «Пауза / Возобновить / Удалить».

**ML-IMPL-2 — has_enough_data** — перед любым ML-расчётом нужно проверять наличие данных. Функция смотрит сколько месяцев с ненулевыми транзакциями есть у пользователя. Если < 2 — прогноз невозможен, показывать заглушку.

**ML-1 — Прогноз** — две модели в зависимости от объёма данных. При 2-5 месяцах — линейная регрессия (`sklearn.LinearRegression`). При 6+ — ARIMA из `statsmodels` (точнее улавливает сезонность). Результат: список прогнозных значений расходов на 3 месяца вперёд.

**ML-2 — Классификация** — если пользователь при вводе транзакции написал описание («Пятёрочка», «Яндекс.Такси»), бот/приложение должен предложить категорию автоматически. Начать со словаря ключевых слов, потом опционально обучить `TfidfVectorizer + LogisticRegression` на накопленных данных пользователя.

**ML-3 — Z-score аномалии** — улучшенная версия базовой из `calculations.py`. Для каждой категории считаем Z-score = (текущее − среднее) / стандартное_отклонение. Если Z > 2.0 — аномалия. Возвращать уровень серьёзности: normal / medium / high.

**ML-4 — Рекомендации** — агрегируем результаты всех предыдущих анализов (аномалии, норма сбережений, подписки, цели) и генерируем 2-3 текстовых рекомендации. Не просто «вы тратите много», а конкретно: «На кафе потрачено на 80% больше обычного — если сократить на 3000₽, цель будет достижима».

**ML-5 — Исчерпание бюджета** — считаем среднедневные расходы за последние 30 дней и делим текущий баланс на это число. Получаем «через N дней баланс станет нулевым». Показывать только если результат < 30 дней — тогда это реальное предупреждение.

**ML-IMPL-3 — Кэш** — ML-расчёты не должны запускаться при каждом открытии страницы аналитики. Нужен механизм кэширования с временем жизни ~1 час. Реализацию передать Team Lead — он интегрирует в `page.data`.
