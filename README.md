## О проекте

**FinControl** — это не просто трекер расходов. Это **3-в-1: аналитик, мотиватор и симулятор**. Приложение не только показывает, сколько вы потратили — оно предупреждает о списаниях, помогает планировать цели и моделирует сценарии «что, если я куплю...».

Один Python-кодbase покрывает iOS и Android одновременно: Flet транслирует Python-код в нативный Flutter UI под капотом. Без Swift, без Kotlin, без React Native.

## Возможности

### Доходы и расходы
- Добавление регулярных доходов (зарплата) и разовых поступлений
- Быстрый ввод расходов прямо с главного экрана
- Категории расходов: Еда, Транспорт, Здоровье, Покупки, Развлечения, Жильё, Образование и другие
- История всех транзакций с фильтрацией по типу

### Финансовые цели
- Создание цели с суммой и дедлайном
- Автоматический расчёт необходимого ежемесячного взноса
- Пополнение цели — деньги реально списываются с баланса
- Прогресс-бар, темп накоплений, визуальное достижение 100%

### Управление подписками
- Учёт всех платных подписок (ежемесячных и ежегодных)
- Расчёт суммарной стоимости в месяц
- Дата следующего списания для каждой подписки
- Умные рекомендации: дубликаты сервисов, годовая стоимость вместо месячной

### Аналитика
- Графики доходов и расходов за выбранный период
- Детализация по категориям
- Прогноз финансов на основе текущей динамики
- Симуляция: «Что изменится в бюджете, если я куплю X?»

### Telegram-бот
- Быстрый ввод транзакций командами: `+5000 зарплата` / `-300 кафе`
- Уведомления о предстоящих списаниях по подпискам
- Напоминания об откладывании на цели
- Быстрая аналитика командой `/stats`
- Авторизация и привязка аккаунта

## Архитектура

```
fincontrolapp/
├── main.py              # Точка входа, навигация, авторизация
├── database.py          # SQLite: схема таблиц, соединение
├── db_queries.py        # Все запросы к БД (репозитории)
├── session.json         # Сохранённая сессия (user_id)
├── components/
│   ├── base_page.py     # Базовый класс экранов (Template Method)
│   └── theme.py         # Цвета и стили (AppTheme)
├── pages/
│   ├── auth.py          # Авторизация / регистрация
│   ├── home.py          # Главный экран
│   ├── transactions.py  # Все транзакции с фильтром
│   ├── income.py        # Доходы
│   ├── expenses.py      # Расходы по категориям
│   ├── goals.py         # Финансовые цели
│   ├── subscriptions.py # Подписки
│   ├── analytics.py     # Аналитика и графики (Пока тут)
│   └── settings.py      # Настройки
└── assets/              # SVG-фоны и иконки навигации
```

### Ключевые паттерны

- **Template Method** — `BasePage` задаёт структуру экрана (`build_header` + `build_body`), каждая страница реализует `build_body()`
- **Repository** — весь SQL инкапсулирован в `db_queries.py`, UI не знает про базу данных
- **Lightweight DI** — глобальное состояние через `page.data`: `user_id`, `navigate()`, `logout()`


## Установка и запуск

### Требования
- Python 3.10+
- [Flet](https://flet.dev) (`pip install flet`)

### Запуск десктопной версии

```bash
git clone https://github.com/your-org/fincontrol.git
cd fincontrol
pip install flet
python main.py
```

### Сборка для мобильных платформ

```bash
# Android
flet build apk

# iOS 
uv run flet run --ios main.py
```


## База данных

SQLite, схема создаётся автоматически при первом запуске.

| Таблица | Назначение |
|---|---|
| `users` | Пользователи (email, phone, telegram_id, password_hash) |
| `categories` | Категории доходов и расходов |
| `transactions` | Все операции (доходы и расходы) |
| `goals` | Финансовые цели |
| `subscriptions` | Подписки с датами списания |

Пароли хранятся в виде `pbkdf2_hmac(sha256, password, salt, 100_000_iterations)` — без открытого текста.

## Безопасность

- Хэширование паролей: PBKDF2-HMAC-SHA256 с уникальной солью на каждого пользователя
- Параметризованные SQL-запросы во всех функциях `db_queries.py` (защита от инъекций)
- Данные хранятся локально на устройстве
