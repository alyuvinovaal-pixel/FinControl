## Используемые библиотеки и зависимости

### Встроенные модули Python (не требуют установки)

| Модуль | Назначение |
|--------|------------|
| **os** | Стандартный модуль для работы с операционной системой. Используется для чтения переменных окружения (токена) через `os.getenv()`. |
| **asyncio** | Стандартный модуль для асинхронного программирования. Позволяет боту одновременно обрабатывать множество запросов, не блокируя выполнение программы. |

### Импортируемые классы из aiogram

| Класс/Функция | Откуда импортируется | Назначение |
|---------------|---------------------|------------|
| **Bot** | `from aiogram import Bot` | Главный класс бота. Отвечает за взаимодействие с Telegram API, отправку сообщений, файлов, запросов. |
| **Dispatcher** | `from aiogram import Dispatcher` | Диспетчер сообщений. Получает все обновления от Telegram и направляет их в нужные обработчики. |
| **types** | `from aiogram import types` | Пакет с типами данных Telegram. Содержит классы: Message, CallbackQuery, User, Chat и другие. |
| **Command** | `from aiogram.filters import Command` | Фильтр для команд. Позволяет ловить сообщения, начинающиеся с /start, /help и т.д. |

### Устанавливаемые зависимости (из requirements.txt)

| Библиотека | Версия | Назначение |
|------------|--------|------------|
| **aiogram** | 3.18.0 | Асинхронный фреймворк для создания Telegram-ботов. Содержит всё необходимое для работы с API Telegram. |
| **python-dotenv** | 1.0.1 | Загружает переменные окружения из файла `.env`. Используется для безопасного хранения токена бота отдельно от кода. |

### Почему выбраны эти библиотеки

#### aiogram 3.x
- **Асинхронность** — бот не блокируется при ожидании ответов от Telegram, может одновременно обслуживать много пользователей
- **Современность** — актуальная версия, активно развивается, хорошая документация
- **Фильтры** — удобная система фильтрации сообщений (по командам, тексту, регулярным выражениям)
- **Состояния** — встроенная поддержка FSM (Finite State Machine) для многошаговых диалогов

#### python-dotenv
- **Безопасность** — токен не попадает в Git (если `.env` добавлен в `.gitignore`)
- **Удобство** — не нужно править код при смене токена или добавлении других секретов
- **Стандарт** — общепринятый способ хранения конфигурации в Python-проектах
